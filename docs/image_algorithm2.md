# 方案 B：MODNet + 径向 Alpha 融合

## 概述

方案 B 是项目中的备选方案，核心目标是：

> 用高质量 alpha matte 精准限定融合区域，尽量只动面部附近，避免身体或场景物体被错误扭曲。

相比方案 A 的"整体融合感"，方案 B 更强调"局部可控性"。

这套方案更适合：

- 你只想换脸，不想动身体或场景物体
- 图 1 和图 2 的身体结构差异很大
- 背景里有船、花、手臂、衣摆等不希望被扭曲的区域

---

## 核心思路

方案 B 的处理链路是：

```text
MODNet 抠图 → BiSeNet 面部定位 → 径向 Alpha Mask → 边缘色调匹配 → 直接 Alpha Blending
```

每一步解决的问题：

1. **MODNet 抠图**：得到高质量连续 alpha matte
2. **BiSeNet 面部定位**：找到面部中心坐标
3. **径向 Alpha Mask**：从面部中心向外衰减，限制融合范围
4. **边缘色调匹配**：消除融合边缘的色差
5. **直接 Alpha Blending**：简单、稳定、无色调漂移

---

## 1. MODNet 人像抠图

MODNet 是专门做人像 matting 的模型，和 BiSeNet 的定位完全不同：

- BiSeNet 输出的是 **19 类语义标签**（皮肤、眼睛、头发等），适合"理解面部结构"
- MODNet 输出的是 **连续 alpha matte**（0~255），适合"知道每个像素有多少比例属于前景"

MODNet 的优势在于：

- 能保留发丝、半透明头发边缘等细节
- 输出值是连续的，天然适合做 alpha blending
- 不需要 trimap 等额外输入，纯 RGB 即可
- 模型仅 25MB，推理速度快

在方案 B 中，MODNet 负责提供精确的人像边界，而不是面部结构理解。

---

## 2. 为什么不用整个人像 alpha 直接融合

这是方案 B 设计中最关键的决策之一。

如果直接把 warp 后的整个人像 alpha 拿来做 `result = src * alpha + dst * (1 - alpha)`，会出现典型问题：

- 图 1 中有的身体、衣服、手、物体
- 图 2 对应位置未必有这些结构
- 仿射变换对非面部区域（船体、衣摆、手臂）会产生严重变形

实际效果就是：船体扭曲、身体崩坏、衣摆异常。

所以方案 B 不直接用 MODNet alpha 全量融合，而是进一步做"**面部中心径向裁剪**"。

---

## 3. 径向 Alpha Mask

这是方案 B 的核心设计：从面部中心构建径向衰减 mask。

### 步骤

1. 用 BiSeNet 辅助检测面部区域，计算面部中心坐标 `(cx, cy)`
2. 根据面部大小计算融合半径（默认 = 面部大小 * 1.8）
3. 构建欧氏距离场
4. 用 smoothstep 曲线（`3t^2 - 2t^3`）将距离映射到 alpha 权重
5. 与 MODNet alpha 相乘，得到最终融合 alpha

### 效果

```text
面部中心附近：alpha ≈ 1.0（完全融合图 1）
离面部越远：  alpha 平滑衰减
超出半径：    alpha = 0（完全保留图 2）
```

### 为什么用 smoothstep 而不是线性衰减

线性衰减在起始点和终止点会有导数不连续，视觉上表现为可见的过渡线。

smoothstep（`3t^2 - 2t^3`）在两端的导数为 0，过渡更柔和，不会产生可见的过渡环。

### 为什么用径向而不是垂直切线

项目早期曾尝试过"垂直衰减"（人物上 40% 融合，下 60% 保留底图），但这种方式会在肩膀水平线处产生明显的水平色差带。

径向衰减从面部中心向各方向均匀扩散，不会产生方向性的硬切线。

---

## 4. 边缘色调匹配

即使 alpha 过渡足够平滑，如果图 1 和图 2 在融合边缘的色调差异很大，仍然会看到色差。

原因：`src * 0.3 + dst * 0.7` 在 alpha=0.3 处，如果 src 偏暖、dst 偏冷，混合结果会是一个不自然的中间色。

### 解决方案

在 alpha blending 前，对图 1 的 warped 区域做一次局部色调匹配：

1. 在 alpha 处于 5%~40% 的环带区域采样
2. 分别统计 src 和 tgt 在该环带内的 LAB 均值和标准差
3. 用统计匹配将 src 的色调拉近 tgt
4. 保守混合（50% 迁移），避免过度偏色

这样在"交汇边缘"处，两张图的色调已经接近，alpha 混合后就不会出现色差。

---

## 5. 为什么不用拉普拉斯金字塔

方案 B 最早曾尝试过拉普拉斯金字塔分频融合，但遇到了几个问题：

### 低频 mask 扩散

拉普拉斯金字塔的 mask 通过 `cv2.pyrDown` 逐层构建。pyrDown 本质是高斯模糊 + 下采样，5 层 pyrDown 后，一个点状的非零值会扩散到 32 像素范围。

这意味着即使原始 alpha 在边缘是 0，低频层的 mask 在那些位置可能是 0.1~0.3，导致前景的色调在边缘外"泄漏"。

### 频段色调漂移

分频融合在每个频段独立混合，然后重建。如果两张图在某个频段的能量分布差异很大，重建后会出现不自然的色调偏移。

### 最终选择

放弃金字塔，改用：

> **MODNet 连续 alpha + 径向 mask + 色调预匹配 + 直接 alpha blending**

原因很简单：

- MODNet 本身已经给出了高质量的柔性边缘
- 径向 mask 已经提供了平滑的空间衰减
- 再做复杂的频段分解，反而引入额外的不可控因素
- 直接 alpha blending 最简单、最稳定、最可预测

---

## 6. Warp 边缘黑线处理

`cv2.warpAffine` 变换后，图像有效区域边缘的像素默认用 0（黑色）填充。当 alpha > 0 时，这些黑色像素会以 `黑色 * alpha` 混进结果，产生细黑线。

解决方案：

1. `warp_image` 使用 `borderMode=cv2.BORDER_REFLECT` 填充边界
2. 对 warped alpha 做 5x5 erode 内缩，去掉最外圈 2px 的 alpha 残留

---

## 性能特点

方案 B 相比方案 A 有几个性能优势：

- **无泊松融合**：不需要求解拉普拉斯方程，计算量小
- **直接 alpha blending**：纯矩阵运算，numpy 广播即可完成
- **色调匹配只在 ROI 内**：不做全图 LAB 转换
- **MODNet 推理仅 25MB**：比 BiSeNet 更轻量

但方案 B 也有额外开销：

- 需要同时运行 MODNet（抠图）和 BiSeNet（面部定位）
- 径向距离场构建需要 `np.mgrid` 生成全图坐标

---

## 优点

- 融合范围精确可控
- 不会影响身体、船体、衣摆等非面部区域
- 径向 smoothstep 过渡自然，无方向性硬切线
- alpha blending 简单稳定，无频段漂移
- MODNet 发丝级边缘质量

## 缺点

- 没有泊松融合的梯度域匹配，边缘自然度略逊
- 依赖色调预匹配质量
- 如果面部中心定位不准，径向 mask 会偏移
- 不适合需要整个人物都融进场景的情况

---

## 推荐使用场景

推荐优先使用方案 B 的场景：

- 只想换脸，不想动身体或周围物体
- 图 1 和图 2 的身体结构差异很大
- 场景中有船、花、手臂等不希望被变形的物体
- 更看重结果的可控性和可预测性

如果你更在意"整体融合感"和"边缘连续性"，则更建议看方案 A。
